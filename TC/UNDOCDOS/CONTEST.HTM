The current contest, attrib, is very similar to the DOS utility of the same name. You write a program that will 'switch' the attributes of a file for the attribute(s) specified.
Here are the rules:

The Objective:
To get the smallest executable file obeying all of the rules below.

The Rules:
  1. You can assume the screen will be in text mode 03h. (though not applicable here)

  2. Your program will assume the following will be on the command line:
    filename.ext ar
    Where the a and the r specify to 'switch' the archive and/or readonly attributes. There can be any kind of combination of these two and can be more than one of the same letter on the command line.
    For instance, if there is an even amount of r's on the command line, the readonly attribute must be unchanged. Same with the a for archive. However, if there is an odd amount of a letter on the command line, this attribute must switch from on to off or off to on.
  3. Note: If there is not an a on the command line, the archive attribute must be unchanged. Same for the r (readonly) attribute.
  4. You can assume there will be at least one letter (either 'a' or 'r') on the command line. (all lowercase letters)
  5. The filename will be before the a or the r and there will be only one space between the filename and the first letter. However there will be no spaces between the letters.
  6. Your code can not crash the computer in any way. It must start and exit leaving the system "stable".

  7. You can assume that the command line will be entered correctly.


The Rules For Your Submission:
You must include all source(s) and the executable with your submission.
  1. If your email software does not have an attachment utility, include your code at the bottom of your email message (do not attached executable in this way).

  2. You must include comments at the top of your code in the following format: (variants are OK)
        NAME: your name or alias. If no name is given here, I use up to 8 chars of your first or last name.
        WEB PAGE: If you have a Web Page, include it here and I will include it with your entry below
        DATE: the date submitted
        ASSEMBLER: the "brand" and version of the assembler
        SPECIAL NOTES: any special notes on how to assemble or execute your submission
        YOUR CODE starts here.

  3. Your code must be for DOS 5.0 or before, on an INTeL based CPU and can use any advantages/commands/services (following the rules above) up to and including the 80386. You can NOT use any instructions/services of the 80486 and higher.

  4. You must include comments with your code. If you do something "unreal", comment it. Don't assume anything. You must verify (with comments) everything "unreal" you do.



Notes/FAQs:
Feel free to ask any questions pertaining to the contest.
